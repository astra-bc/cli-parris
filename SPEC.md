## 1. 概要

| 項目 | 内容 |
|------|------|
| ゲーム名 | Nononono（灰都殺伐代行） |
| ジャンル | ローグライク × タイミングアクション（パリィ） |

## 2. ゲームフロー

```
起動
  │
  ▼
タイトル画面
  │  [W] で開始 / [Q] で終了
  ▼
戦闘画面（Floor 1 固定）
  │  敵3体と戦闘
  │
  ├─ 全敵撃破 → 勝利画面 → [R] リスタート / [Q] 終了
  └─ HP 0     → 死亡画面 → [R] リスタート / [Q] 終了
```

### 2.1 タイトル画面

- NONONONOのASCIIアートロゴ表示
- 「灰 都 殺 伐 代 行」タイトル表示
- 操作説明・パリィの心得を表示
- `W` キーで戦闘開始、`Q` で終了
- 全角入力を検出した場合、警告メッセージを1秒間表示
- macOSでは起動時にAppleScriptでASCII入力モードへ自動切替を試行

### 2.2 戦闘画面

- 上部: ゲームタイトルとフロア番号（Floor 1 固定）
- 中部: 敵一覧（各敵に名前行 + 攻撃バー行）
- 下部: プレイヤーステータス、操作ガイド、統計情報

### 2.3 勝利画面

- 「VICT」ASCIIアート表示（黄色・太字）
- 「灰は燃え尽き、都は静寂に還る」メッセージ
- 戦績表示: 残りHP、最大コンボ、総パリィ数、PERFECT/Good/MISS回数

### 2.4 死亡画面

- ドクロのASCIIアート表示（赤色）
- 「大いなる死に賛歌を！」メッセージ
- `R` でリスタート、`Q` で終了

## 3. 操作体系

| キー | アクション | 備考 |
|------|-----------|------|
| `W` | パリィ：上段 | 攻撃硬直中は無効 |
| `A` | パリィ：左 | 攻撃硬直中は無効 |
| `S` | パリィ：下段 | 攻撃硬直中は無効 |
| `D` | パリィ：右 | 攻撃硬直中は無効 |
| `↑` | ターゲットを上に切替 | 循環（先頭→末尾） |
| `↓` | ターゲットを下に切替 | 循環（末尾→先頭） |
| `Space` | 攻撃 | 硬直中・クールダウン中は無効 |
| `Enter` | 残火の血霧 | 残火MAX＋敵HP≦1が条件 |
| `R` | リスタート | ゲームオーバー/クリア時のみ |
| `Q` | 終了 | 全画面で有効 |

- 大文字・小文字の区別なし（W/w どちらでも可）
- キーコード127超（全角入力）はフラッシュして無視（↑↓は除外）

## 5. 武器

現バージョンでは固定装備。選択不可。

| パラメータ | 値 |
|-----------|-----|
| 名前 | 殺意の出刃包丁 |
| ATK | 1 |
| 攻撃硬直 | 0.5秒 |
| 攻撃クールダウン | 2.0秒 |

### 5.1 攻撃の挙動

- `Space` で発動。硬直中・クールダウン中は発動不可
- ターゲット中の敵にのみ有効
- ダメージ = `WEAPON.atk`（= 1）
- **●マークが残っている敵**: HPは **1 まで** しか下がらない（`max(1, hp - dmg)`）
- **●マークが 0 の敵**: HPは **0 まで** 下がる（`max(0, hp - dmg)`）
- 攻撃後、攻撃硬直（0.5秒）が発生 → その間パリィ不可
- 攻撃後、攻撃クールダウン（2.0秒）が発生 → その間再攻撃不可
- ダメージメッセージを1.0秒間表示

### 5.2 攻撃クールダウンゲージ

- 画面下部に `ATK [██████████] READY!` / `ATK [████░░░░░░] X.Xs` で表示
- クールダウン中はゲージが徐々に充填される描画

## 6. パリィシステム

### 6.1 攻撃バー

- バー全長: 40文字（`BAR_LEN = 40`）
- `●` が左端（位置0）から右端（位置39）へ等速で移動
- 移動時間（秒）は敵ごとの `speed_min` 〜 `speed_max` の範囲でランダム決定
- 敵の HP が 1 のとき、移動時間が **2倍** に減速

### 6.2 パリィゾーン

- PERFECT位置: 位置35（`PERFECT_POS = 35`）
- Goodゾーン: `PERFECT_POS - parry_zone` 〜 `PERFECT_POS + parry_zone`
- ゾーン幅は敵ごとに異なる（`parry_zone` パラメータ）
- ゾーン外観: `░` で表示、PERFECT位置は `█` で表示

### 6.3 パリィ判定

パリィは **ターゲット中の敵** に対してのみ有効。攻撃硬直中はパリィ不可。

#### 方向判定（先に評価）

- 各攻撃にはランダムな方向（W/A/S/D）が割り当てられる
- 画面上に `[W]` `[A]` `[S]` `[D]` で表示
- 押したキーと方向が **不一致** → 即 **MISS** 判定（タイミング不問）

#### タイミング判定（方向一致時）

| 判定 | 条件 | 効果 |
|------|------|------|
| **PERFECT** | `●` の位置 == `PERFECT_POS`（位置35） | 残火 +4、コンボ +1、パリィ数 +1 |
| **Good** | `●` の位置が Goodゾーン内 | 残火 +1、コンボ +1、パリィ数 +1 |
| **MISS** | 上記以外 | 敵ATK分のダメージ、残火 -2、コンボ 0 |

### 6.4 攻撃の自動MISS

- `●` がバー右端（位置39）に到達すると自動的に **MISS** 判定
- 敵のATK分のダメージ、残火 -2、コンボリセット

### 6.5 攻撃バーのリセット

- パリィ判定後（PERFECT/Good/MISS）、敵ごとの `cooldown` 秒経過で次の攻撃バーが開始
- 次の攻撃は速度・方向が再ランダム化される

### 6.6 バーの色分け

- ターゲット中の敵の `●`: **赤色・太字**
- ターゲット外の敵の `●`: **黄色**
- PERFECT結果: **黄色・太字**
- Good結果: **緑色**
- MISS結果: **赤色**

## 7. 残火ゲージ

| 操作/イベント | 変化量 | 備考 |
|-------------|--------|------|
| PERFECT パリィ | +4 | 上限10でキャップ |
| Good パリィ | +1 | 上限10でキャップ |
| MISS（方向ミス含む） | -2 | 下限0でキャップ |
| 攻撃バー到達（自動MISS） | -2 | 下限0でキャップ |
| 残火の血霧発動 | → 0 | 全消費 |

## 8. 撲殺システム（残火の血霧）

### 8.1 発動条件

以下の **全て** を同時に満たすとき、`Enter` で発動可能:

1. 残火ゲージが **MAX**（10/10）
2. ターゲット中の敵の **●マークが 1 以上**
3. ターゲット中の敵の **HP が 1 以下**

### 8.2 発動効果

- 残火ゲージを **0** にリセット
- ターゲット中の敵の **●を 1 つ削る**
- ●が **残っている** 場合: 敵の HP を **最大値まで全回復**、メッセージ2秒表示
- ●が **0 になった** 場合: 敵の HP を **0** にして **撲殺**（完全撃破）、メッセージ2秒表示

## 9. 敵

### 9.1 共通仕様

- 最大同時出現数: 3体（現バージョン固定）
- 各敵は独立した攻撃バーを持つ（同時に攻撃してくる）
- 死亡した敵は `--- DEAD ---` と薄暗く表示、攻撃バー停止
- ターゲット中の敵は名前行の先頭に `|` を表示、太字

### 9.2 敵パラメータ一覧

| パラメータ | 渋谷の不動産屋の鼬 | (ボス) 堕落した都知事 | 池袋の町中華の豚 |
|-----------|-------------------|---------------------|-----------------|
| AA | `(^w^)` | `{o_o}` | `(-.-)`|
| 難易度表示 | ★☆☆ | ★★★ | ★★☆ |
| HP | 6 | 20 | 8 |
| ATK | 3 | 8 | 5 |
| ●マーク | 1 | 3 | 1 |
| バー速度（秒） | 2.1 〜 2.9 | 3.5 〜 5.0 | 7.0 〜 10.0 |
| 攻撃後クールダウン | 1.5秒 | 0.5秒 | 2.0秒 |
| パリィゾーン幅 | 6（広い） | 1（極狭） | 4（やや広い） |
| 攻撃方向 | W/S/A/D | W/S/A/D | W/S/A/D |

### 9.3 表示順序

上から順に:
1. 渋谷の不動産屋の鼬
2. (ボス) 堕落した都知事
3. 池袋の町中華の豚

### 9.4 HP1 時の減速

- 敵の HP が 1 のとき、攻撃バーの移動時間が **2倍** になる（速度半減）
- 残火の血霧を撃つ猶予を確保するための仕様

## 10. コンボシステム

- PERFECT または Good のパリィ成功で **コンボ +1**
- MISS（方向ミス・タイミングミス・自動MISS）で **コンボ 0** にリセット
- 最大コンボ数を記録
- 画面下部に `COMBO:X  MAX:X  PARRY:X` で表示

---

## 11. 統計情報

画面下部にリアルタイム表示:

- `COMBO`: 現在のコンボ数
- `MAX`: 最大コンボ数
- `PARRY`: 総パリィ成功数（PERFECT + Good）
- `PERFECT`: PERFECT回数
- `Good`: Good回数
- `MISS`: MISS回数

勝利画面では `残りHP`、`最大コンボ`、`総パリィ`、各判定回数を表示。

## 13. 勝利・敗北条件

| 条件 | 結果 |
|------|------|
| 3体全ての敵のHPが0以下 | **勝利** |
| プレイヤーHPが0以下 | **敗北（パーマデス）** |

- 勝利/敗北中は全入力を無視（`R`/`Q` のみ受付）
- プレイヤー死亡時、敵の攻撃バーは停止するがバー自体は表示継続

## 14. 技術仕様

### 14.1 メインループ

- `time.sleep(0.016)` で約60fps
- `stdscr.nodelay(True)` でノンブロッキング入力
- 毎フレーム `stdscr.erase()` で全画面再描画

### 14.2 入力処理

- macOS起動時: AppleScript経由でASCII入力モードへ切替試行
- 全角入力検出: キーコード > 127（↑↓キーを除外）の場合 `curses.flushinp()` でバッファクリア
- タイトル画面で全角入力を検出した場合、警告メッセージを表示

### 14.3 パッケージ構成

```

│   ├── __init__.py      # バージョン定義
│   ├── __main__.py      # python -m cli_parris 用
```
